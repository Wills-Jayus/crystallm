1) 背景
- 用户请求：读取最新实验结果目录 `/root/autodl-tmp/model/CrystaLLM/experiments/na2cl2_qwen_v4_small`，分析“生成不通过结构”的原因，深入解释失败链条。

2) 目标
- 解释每轮结构为何未通过（validation 不通过）的主要原因与证据链。
- 针对 `space_group_check_error / symmetry_dataset_unavailable` 给出可复现的根因（含 spglib 真实错误信息）。
- 产出一份可读的报告落盘到 docs，并对代码做必要的可解释性增强。

3) 结论摘要（关键发现）
- Round 2/3 大面积失败的“决定性”原因不是 ALIGNN 打分器，而是 Llama/Qwen 的“提示改写”导致 `data_` 行出现小数点：`data_Na2.1Cl1.9`。
  - 当前 `extract_data_formula` 只允许 `[A-Za-z0-9]+`，遇到 `.` 会直接报 “could not find data_”，从而让 `formula_check` 与 `strict_valid` 连锁失败（Round 2/3 `validation_stats.passed=0`）。
- Round 1 的失败主要来自“生成体系跑偏”（data 行写 Na2Cl2，但实际 `_chemical_formula_sum` 等变成 Ba/Cu/Si/O 等），以及键长合理性不足、空间群一致性不满足等。
- `symmetry_dataset_unavailable` 并非总是“空间群完全算不出”，而是 spglib 在某些输入下返回 dataset=None。经本机复现，核心触发原因是 spglib 错误：`too close distance between atoms`（结构展开后产生极近/重叠位点，symmetry dataset 直接不可用）。
- 为提升可解释性，我们对 `crystallm/_metrics.py` 做了增强：当 dataset=None 时，把 spglib 错误信息 + 碰撞尺度（min_neighbor_dist）写入 error 字段，便于定位。

4) 输出物（落盘）
- 已生成报告（更聚焦 space_group_check_error 根因与修复建议）：
  - `model/CrystaLLM/docs/report_symmetry_dataset_unavailable_na2cl2_qwen_v4_small.txt`

2.2 解析失败根因：extract_data_formula 的正则不接受小数点
- 读取：`model/CrystaLLM/crystallm/_utils.py`
- 关键代码：
  - `extract_data_formula(cif_str)` 使用 `re.search(r"data_([A-Za-z0-9]+)\\n", cif_str)`
  - 因此 `data_Na2.1Cl1.9` 不匹配，抛异常：
    - `Exception: could not find data_ in: ...`
- 影响链条（由 validate/is_valid 调用关系导致）：
  - `validate_cif` -> `is_formula_consistent` -> `extract_data_formula` 抛异常 => formula_ok=False + 记录 formula_check failed
  - `strict_valid` -> `is_valid` -> `is_formula_consistent` 再次触发同样异常 => strict_valid=None + 记录 strict_valid failed
  - 最终 Round2/3 大面积 `validation_ok=false`，并在报告中表现为 passed=0。

2.3 Round2/3 还伴随“体系跑偏”
- 读取示例：`round_02/cifs/sample_8.cif`
  - data 行是 Na/Cl 小数，但 `_chemical_formula_sum` 变成 Sr/Mg/Cl（非目标体系），进一步导致 composition 相关判定失败。

3) 重点问题：space_group_check_error / symmetry_dataset_unavailable 的真实含义与复现
3.1 先澄清概念
- 在 `crystallm/_metrics.py` 的 `space_group_consistency_details()` 中：
  - `consistent=True`：声明空间群 == 检测空间群
  - `consistent=False`：两者都有值但不一致（即“space group inconsistent”）
  - `consistent=None`：无法判定（解析/对称性分析失败），此时 error 字段会包含原因（历史版本里是 `symmetry_dataset_unavailable`）。

3.2 环境与导入排查（避免“同名包干扰”）
- 发现系统 Python3.13 无 pymatgen；项目实际使用 conda env `crystallm_env`（Python3.10）。
- conda 中 `crystallm` 是 editable 安装，实际映射到：
  - `/root/autodl-tmp/model/CrystaLLM/crystallm`
- 同时 workspace 下存在 `/root/autodl-tmp/crystallm/cif_cleaning.py`（非完整包），在 cwd=/root/autodl-tmp 时会干扰 import。
- 解决方式：在复现脚本中 `cd /root/autodl-tmp/model/CrystaLLM` 再运行，确保导入的 `crystallm` 来自正确目录。

3.3 复现（抓取 spglib 真实错误信息）
复现脚本核心做法：
- 对样本 CIF：
  1) 直接 `Structure.from_str(cif)` 并跑 SpacegroupAnalyzer（对照）
  2) 先执行 `replace_symmetry_operators(cif, 'P4/mmm')`（模拟 clean_cif 的行为），再跑 SpacegroupAnalyzer
  3) 同步调用 `spglib.get_symmetry_dataset(cell, symprec=...)` 并读取 `spglib.get_error_message()`

关键复现结果（symprec=0.1）：
- sample_1（替换后）：
  - pymatgen dataset=None
  - spglib dataset=None
  - spglib error message：`too close distance between atoms`
  - 碰撞尺度：min_neighbor_dist<0.1Å≈0.0300Å
- sample_5（替换后）：
  - spglib error message：`too close distance between atoms`
  - 碰撞尺度：min_neighbor_dist<0.1Å≈0.0805Å
- sample_6（通过对照）：
  - dataset 正常，检测空间群 `P4/mmm`。

3.4 几何体检（替换对称操作前后最小原子间距）
- sample_1：
  - before_replace：nsites=5,  min_dist≈1.3872Å
  - after_replace ：nsites=44, min_dist≈0.0300Å
- sample_5：
  - before_replace：nsites=9,   min_dist≈0.5479Å
  - after_replace ：nsites=118, min_dist≈0.0805Å
- sample_6：
  - before/after：nsites=4, min_dist≈2.9567Å（稳定）

结论：`symmetry_dataset_unavailable` 的“物理根因”是结构展开后产生极近/重叠位点，spglib 直接拒绝给 dataset。

3.5 验证一个可行的“碰撞恢复”思路（实验验证）
- 对替换后产生碰撞的结构，执行：
  - `Structure.merge_sites(tol=0.1, mode='delete')`
- 观察：
  - sample_1 替换后结构在 merge_sites(tol=0.1) 后，spglib 在 symprec=0.1 下恢复正常（error 变为 no error，能检测到空间群）
- 说明：merge_sites 可作为“可选修复”策略（是否落地需进一步评估对物理意义的影响）。

备注（重要）：
- 本次代码落地的是“错误信息增强”（见下一节），并未在代码中自动执行 merge_sites 修复；

1) 对“未通过结构”的整体归因（按影响面排序）
- A. Prompt 被 Llama/Qwen 改为小数 data 行（`data_Na2.1Cl1.9`）+ `extract_data_formula` 不支持 `.`：
  - 直接导致 Round2/3 `formula_check` 与 `strict_valid` 连锁失败
  - 还会诱发生成分布漂移（更多非目标元素体系）
- B. Round1 生成体系跑偏（data 行写 Na2Cl2，但 `_chemical_formula_sum`/`_structural` 等变成其他材料体系）：
  - 引发 composition inconsistent
- C. 几何/晶体学质量问题（键长不合理、空间群不一致、或 spglib 因碰撞导致 dataset=None）：
  - 需要更鲁棒的后处理/约束策略

2) 建议的工程方案（分层）
生成侧（Prompt/策略）：
- 禁止/避免 `data_` 行出现小数点或非标准字符；强制使用可被解析的配方标识。
- 加强提示约束：同时给 `_chemical_formula_sum`/模板位点/Wyckoff 约束，减少体系跑偏与几何噪声。

过滤与验证侧（代码/鲁棒性）：
- 已落地：当 spglib dataset 不可用时，输出具体错误原因（too close 等）与碰撞尺度。
- 建议评估落地：
  - 若 spglib_err=too close，可选做 `merge_sites(tol=symprec)` 再重试；
  - 或实施 symprec 扫描/重试策略（在不破坏物理意义前提下）。

评分侧（ALIGNN）：
- ALIGNN 输出包括：`ok`、`error/errors`、每个 property 的数值（formation_energy、bandgap）。
- 评分器整体未发现导致 Round2/3 全失败的主因；主要失败发生在过滤/验证链路（formula/strict_valid）。

3) 后续建议的验证清单
- 修复/约束 prompt 后，重新跑一轮小样本：
  - 检查 Round2/3 `formula_ok` 是否恢复
  - 检查 `space_group_check_error` 是否显著减少，且 error 是否可解释（too close 等）
