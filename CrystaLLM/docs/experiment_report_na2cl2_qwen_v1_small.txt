CrystaLLM + ALIGNN(GPU) + Qwen3(vLLM) 闭环实验报告（na2cl2_qwen_v1_small）
生成时间：2025-12-18

1. 实验目标（做了什么）
--------------------
本实验验证一个三轮闭环流程是否能稳定运行，并将“优化器 LLM”从原先 LLaMA 切换为 Qwen3（通过 vLLM OpenAI API 方式提供服务）。

闭环每一轮的逻辑是：
  (1) 用 CrystaLLM 根据 prompt（少量 CIF 高层行）生成若干 CIF 候选；
  (2) 对每个 CIF 做“清洗 + 物理/晶体学验证”（过滤器），记录哪些结构可信；
  (3) 调用 ALIGNN（ZMQ 服务）对每个 CIF 预测 formation_energy、bandgap 并汇总统计；
  (4) 把“上一轮 prompt + 本轮摘要”发给 Qwen，让 Qwen给出 next_prompt_lines（下一轮 prompt），进入下一轮。

实验输出目录：
  /root/autodl-tmp/model/CrystaLLM/experiments/na2cl2_qwen_v1_small

3. 目录结构与关键产物（每轮都能看到什么）
----------------------------------------
每轮目录（round_01/round_02/round_03）内主要文件：
  - prompt.txt：该轮实际用于 CrystaLLM 生成的 prompt（高层行）
  - cifs/sample_*.cif：CrystaLLM 生成的 CIF 候选（数量=8）
  - cif_quality.json / cif_quality.csv：过滤器对每个 CIF 的清洗+验证结果（是否通过、失败原因等）
  - cif_quality_summary.txt：过滤器统计摘要（通过率、常见失败点等）
  - scores.csv：ALIGNN 预测结果（formation_energy、bandgap 等），并合并了过滤器关键字段
  - summary.json：该轮汇总统计（均值/极值、通过率、Top structures 等）
  - evaluator_summary.json：给优化器 LLM 的“压缩摘要”（关键指标 + top 结构）
  - qwen_output.json：仅在 round_01/round_02 存在（因为 round_03 是最后一轮）

全局文件：
  - final_report.json：全流程汇总 + best_overall
  - best_structures/：复制出来的候选（命名带 round/rank/score）

6. 重点观察与问题定位（本实验暴露了什么）
----------------------------------------
(1) Qwen 输出稳定、可解析、能驱动闭环
  - round_01/round_02 的 Qwen 输出都能生成 next_prompt_lines，并确实影响下一轮 prompt.txt。
  - 使用 relaxed_v0 协议时，不要求严格 JSON/固定模板，降低了“模型不服从导致全盘失败”的风险。

(2) ALIGNN GPU 打分已正常工作
  - 每轮 total/ok 都是 8/8，说明 ZMQ 服务可用且返回数值。

(3) 过滤器通过率偏低，主要卡在“化学计量一致性”
  - round_02 更是 0/8 通过。
  - 常见失败原因包含：
    - composition inconsistent（CIF 内不同字段/位点统计的化学式对不上）
    - atom site multiplicity inconsistent（位点多重度统计出来的原子总数与化学式对不上）
  - 这说明：仅靠 data_ 行提示很难让生成的 CIF 在多个字段上保持自洽；需要后续在 prompt 或后处理上“更强约束/更强修复”。

(4) Qwen 的目标表述存在偏差
  - round_01 的分析中出现“探索更低带隙”的措辞，与当前目标（max bandgap）相矛盾。
  - 但它给出的 next_prompt_lines 仍然可用；后续应在输入提示中更明确地写死目标方向（bandgap 越大越好）。

(5) 少量样本出现验证阶段异常
  - round_01 的部分样本在 validation_reasons 中出现：
      "validation_failed: 'NoneType' object is not subscriptable"
  - 这更像是过滤器实现遇到某些 CIF 边界格式时的异常（导致直接判 invalid），建议后续单独排查这类 CIF 的格式特征并修复验证逻辑的鲁棒性。
