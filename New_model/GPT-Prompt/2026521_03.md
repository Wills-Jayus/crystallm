二、第一阶段：做 skeleton / assignment / cell / coord 精准诊断
====================

目标：
把 unmatched 拆清楚，不再只看最终 match。

请新增脚本：

scripts/diagnose_symcif_v2_failures.py

输入：
- baseline_minprompt eval 结果
- symcif_v2_constrained_fixed_cell eval 结果
- test split manifest
- GT CIF 路径
- generated standard CIFs
- generated SymCIF-v2 records
- Wyckoff lookup

输出：
eval_runs/symcif_v2_diagnosis_202605xx/
  diagnosis_summary.csv
  diagnosis_summary.json
  per_sample_diagnosis.csv
  per_generation_diagnosis.csv
  Log_GPT/symcif_v2_diagnosis_report.md

需要实现以下指标：

1. skeleton_signature

只看 Wyckoff skeleton，不看元素：
例如：
[(2a), (4c), (8f)]
需要统一排序规则，避免顺序差异造成误判。

2. assignment_signature

看元素占据哪个 Wyckoff：
例如：
[(Na,2a), (Cl,2b), (O,8f)]
同样需要规范排序。

3. skeleton@1 / skeleton@5

每个 sample 的前 1 / 前 5 个生成中，是否有 skeleton_signature 与 GT 一致。

4. assignment@1 / assignment@5

每个 sample 的前 1 / 前 5 个生成中，是否有 assignment_signature 与 GT 一致。

5. cell_error

对 formula + SG + skeleton + assignment 已正确的样本，统计：
- a/b/c MAE
- alpha/beta/gamma MAE
- volume relative error
- 按 crystal system 分组统计

6. free_coord_error

对 skeleton + assignment 已正确的样本，比较 free coordinate 误差。
如果 free coordinate 对齐困难，先只统计可明确对齐的 Wyckoff sites。

7. unmatched 分桶

每条 unmatched generation 归因到：
- skeleton_wrong
- assignment_wrong
- skeleton_assignment_correct_but_cell_bad
- skeleton_assignment_correct_but_free_coord_bad
- readable_or_sg_render_issue
- structurematcher_skip_or_timeout
- unknown

8. GT skeleton logprob rank

对于每个 sample：
- 生成合法 skeleton 候选集合；
- 计算 GT skeleton 在候选集合中的 LM logprob rank；
- 如果 GT skeleton 不在候选集合，记录 missing_gt_skeleton。
这一步用于判断问题是 search space 没覆盖，还是 rank 排错。

====================
三、第二阶段：Oracle 诊断实验
====================

目标：
判断如果 skeleton / cell 已知，后续模块到底能做到什么程度。

新增脚本：

scripts/run_symcif_v2_oracle_diagnosis.py

注意：
这是诊断实验，可以使用 GT，但报告必须明确：oracle 不作为正式生成结果。

做三组 oracle：

Oracle A：GT skeleton + generated coord/cell
- 使用 GT Wyckoff skeleton + GT element assignment。
- free coords 和 cell 仍由模型生成。
- 目的：看 skeleton/assignment 如果正确，模型 coord/cell 能否 match。

Oracle B：GT skeleton + GT cell + generated free coords
- 使用 GT skeleton / assignment / cell。
- 只让模型生成 free coordinates。
- 目的：隔离 free coord 问题。

Oracle C：GT skeleton + generated cell vs GT cell
- 使用 GT skeleton / assignment。
- 重点统计模型生成 cell 与 GT cell 的误差。
- 目的：判断 lattice 是否主要拖累 match。

输出：
eval_runs/symcif_v2_oracle_diagnosis_202605xx/
  oracle_summary.csv
  oracle_summary.json
  Log_GPT/symcif_v2_oracle_report.md

报告必须回答：
1. GT skeleton 后 match@5 能提高到多少？
2. GT skeleton + GT cell 后 match@5 能提高到多少？
3. 如果 Oracle A 仍然低，说明 coord/cell 是大问题。
4. 如果 Oracle A 很高，说明主瓶颈是 skeleton/assignment。
5. GT skeleton 在当前模型 rank 中是否靠前？

====================
四、第三阶段：重做 constrained generation：legal skeleton search + proper ranking
====================

目标：
不要再做“盲目 diversity beam”。
改成：
合法 skeleton 搜索 → 完整 LM logprob 打分 → length-normalized ranking → top-k skeleton → 生成 coord/cell。

新增 mode：

symcif_v2_constrained_skeleton_rank

新增脚本：

scripts/sample_symcif_v2_skeleton_rank.py

核心流程：

1. 从 prompt 解析：
- target formula
- target SG

2. 枚举 / beam search 合法 skeleton

状态：
- remaining_counts
- target_sg
- used_nonrepeatable_fixed_sites
- current_skeleton
- cumulative_logprob

合法性约束：
- element 必须来自 remaining_counts > 0；
- Wyckoff letter 必须属于 target_sg；
- multiplicity(SG, letter) <= remaining_counts[element]；
- 不可重复 fixed site 不能重复；
- 选择该 letter 后，remaining_counts 必须仍可由当前 SG 的 multiplicity 集合闭合；
- remaining_counts 全部为 0 时才允许 STOP。

3. scoring

对完整 skeleton 计算完整 LM logprob，不要只用局部启发式分数。

要求：
- 使用 length-normalized logprob；
- 可以记录 raw logprob / normalized logprob；
- 不加入 diversity bonus 作为默认；
- 不加入 arbitrary special-position bias 作为默认；
- 不根据 match 结果调分。

4. ranking

保留 top_k skeleton，例如：
- beam_size = 64
- top_skeletons = 5

5. continuation

对每个 top skeleton：
- fixed coords 由 lookup 自动写；
- free coords 由模型生成；
- cell 用 fixed_cell 版本的生成逻辑；
- 不再写死 beta；
- 按晶系做合法投影；
- volume 根据 cell 参数重算。

6. 输出 n=5 candidates

研发阶段先输出 match@5。

====================
七、第五阶段：决策是否进入 SymCIF-v3
====================

根据诊断结果做决策：

情况 A：
GT skeleton 在 LM rank 中靠前，但当前 sampler 没选到。
结论：继续优化 skeleton search/ranking。

情况 B：
GT skeleton 经常不在合法候选集合里。
结论：lookup / setting / skeleton enumeration 有问题，需要修数据和 converter。

情况 C：
GT skeleton 在候选集合里但 rank 很低。
结论：当前 SymCIF-v2 训练格式没有学好 skeleton prior，需要设计 SymCIF-v3。

SymCIF-v3 方向：
SG + formula
→ Wyckoff skeleton
→ element assignment
→ free coords
→ cell

也就是更接近 CrystalFormer 的 W → A → X → L 顺序。

本轮不要直接实现 SymCIF-v3，只在报告中给出是否需要进入 v3 的判断。
