你现在接手 SymCIF / CrystaLLM 项目。当前任务不是继续修 v2/v3 文本 sampler，也不是直接训练 coords/lattice。请实现下一阶段主线：

SymCIF-v4 = OrbitEngine + Streaming Composition-Exact WA Generator + Orbit-aware Ranker

核心目标：
给定完整 conventional cell formula + fixed SG，生成 formula-exact、SG-legal、可稳定渲染为 CIF 的 Wyckoff-assignment table，最终输出标准 CIF。

硬约束：
1. 输入 formula 必须是完整 conventional cell composition，不是 reduced formula。
2. SG 固定作为输入，不生成 SG。
3. 最终输出必须是 CIF，WA-table 只是中间结构。
4. 不允许用 post-hoc repair 伪装成 generator。
5. 暂时不要训练新的 coords/lattice 模型，除非 OrbitEngine roundtrip 先过线。
6. 不要再优化 v3 文本 sampler、assignment_template_id 分类器、pure rejection、blind diversity。
7. 不要依赖 `pymatgen.Structure.from_spacegroup` 作为核心渲染器；pymatgen/spglib 只能作为 validation/eval 工具。

C. Step 1：实现 OrbitToken / OrbitEngine
============================================================

目标：
建立唯一 symmetry source of truth。后续数据 canonicalization、WA candidate generation、coords interpretation、CIF rendering 都必须用同一套 OrbitEngine。

OrbitToken 至少包含：

```python
@dataclass(frozen=True)
class OrbitToken:
    sg: int
    sg_symbol: str
    hall_number: int | None
    setting_id: str | None

    letter: str
    multiplicity: int
    site_symmetry: str
    enumeration: int | str | None

    representative_expr: tuple[str, str, str]
    free_symbols: tuple[str, ...]
    free_mask: tuple[bool, bool, bool]

    fixed_values: tuple[float | None, float | None, float | None]
    is_fully_fixed: bool

    symmetry_ops: tuple[str, ...]
    origin_shift: tuple[float, float, float] | None
    basis_transform: list[list[float]] | None

    canonical_orbit_id: str
````

OrbitEngine 必须提供：

```python
class OrbitEngine:
    def get_orbits(self, sg: int) -> list[OrbitToken]:
        ...

    def evaluate_representative(
        self,
        orbit: OrbitToken,
        free_params: dict[str, float],
    ) -> tuple[float, float, float]:
        ...

    def expand_orbit(
        self,
        orbit: OrbitToken,
        free_params: dict[str, float],
        symprec: float = 1e-5,
    ) -> list[tuple[float, float, float]]:
        ...

    def render_cif_from_wa_table(
        self,
        wa_table: WATable,
        lattice: dict,
        free_params_by_row: dict,
        formula_counts: dict[str, int],
        sg: int,
    ) -> str:
        ...
```

关键要求：

1. `expand_orbit()` 展开后的 unique site 数必须等于 OrbitToken.multiplicity。
2. 对每个 SG / letter 都要有 multiplicity unit test。
3. 对代表坐标表达式必须正确处理参数关系，例如：

   * x, 2x, z
   * x, -x, z
   * x, x, x
   * fixed fractions like 1/8, 1/4, 1/2
4. 不允许再用“free_mask 直接填 x/y/z”替代 representative_expr。
5. 不允许把 Wyckoff lookup 和 renderer 拆成两套 convention。

优先处理已知问题：

* SG 194 的 12k: x, 2x, z
* SG 227 Fd-3m 的 8a / 16d setting/origin mismatch

如果当前 lookup 和 pymatgen setting 不一致：

* 不要用 `Structure.from_spacegroup` 强行渲染。
* 先用 OrbitEngine 自己的 symmetry operations 展开 full atom loop。
* CIF 中仍写 SG number / symbol，但 atom positions 由 OrbitEngine 展开。

F. Step 4：Streaming exact-cover WA search
=========================================

不要继续先枚举巨大 JSONL。

实现：

src/symcif_v4/wa_search.py
scripts/run_streaming_wa_search.py

核心思想：
每一步选择一个 action：

```text
action = element @ OrbitToken
```

状态：

```python
@dataclass
class SearchState:
    sg: int
    remaining_counts: dict[str, int]
    chosen_rows: list[tuple[str, OrbitToken]]
    used_fixed_orbits: set[str]
    remaining_total_atoms: int
    score: float
```

合法 action 条件：

1. OrbitToken.sg == input SG
2. remaining_counts[element] >= orbit.multiplicity
3. fully fixed orbit 默认不能重复使用
4. free orbit 可重复，但不能导致总 atom count 超目标
5. 扣除该 action 后，remaining_counts 仍然 exact-cover feasible
6. STOP 仅允许在所有 remaining_counts == 0 时触发

必须实现 DP feasibility：

```python
def is_feasible(remaining_counts, available_orbits, used_fixed_orbits) -> bool:
    ...
```

搜索模式：

1. beam search
2. priority search / A*
3. branch-and-bound
4. 可选 learned policy score

不要把所有候选一次性写入巨大 JSONL。输出 topK candidates 即可。

CLI 示例：

```bash
python scripts/run_streaming_wa_search.py \
  --data-root data/structured_symcif_v4 \
  --split test \
  --beam-size 256 \
  --top-k 200 \
  --max-expanded-states 200000 \
  --timeout-per-sample 30 \
  --out-dir reports/symcif_v4_streaming_wa
```

输出：

reports/symcif_v4_streaming_wa/test_streaming_candidates.jsonl
reports/symcif_v4_streaming_wa/search_summary.json
reports/symcif_v4_streaming_wa/search_breakdown_per_sg.csv
reports/symcif_v4_streaming_wa/search_breakdown_per_nsites.csv

指标：

1. candidate_nonempty_rate
2. gt_skeleton_in_topK
3. gt_wa_in_topK
4. mean / median expanded_states
5. timeout_samples
6. truncated_samples
7. per-nsites breakdown
8. per-num_elements breakdown

Gate 2 初步目标：

```text
gt_skeleton_in_top200 >= 99%
gt_wa_in_top200 >= 90%
n_sites >= 6 的 gt_wa_in_top200 明显高于旧 52%-67% 区间
timeout = 0 或可解释
```

```text

```

global feature：

```text

```

模型建议：

版本 1：

* Set Transformer / Transformer encoder without positional encoding
* row tokens as unordered multiset
* pooling -> scalar score

版本 2：

* policy model for streaming search
* 输入 partial state
* 输出 next action score: element@OrbitToken

训练目标至少包含两种：

1. listwise / candidate-set CE：
   positive = GT WA-table
   negatives = same formula+SG generated hard candidates

2. step-level policy CE：
   teacher forcing on canonical GT WA sequence
   每步预测下一个 element@OrbitToken
   使用合法 action mask

hard negatives 必须包含：

1. same skeleton wrong assignment
2. same multiplicity pattern wrong site_symmetry
3. same formula wrong special/general site choice
4. same SG and same nsites but different orbit token
5. high-score but non-GT candidates from streaming search

```text
complex subset n_sites>=6
complex subset num_elements>=4
candidate_nonempty
per-SG breakdown
per-nsites breakdown
```

Gate 2 强目标：

