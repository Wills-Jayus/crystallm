====================
五、skeleton beam 具体逻辑
====================

定义 skeleton：
一个结构的 Wyckoff skeleton 可以表示为：

[(element_1, wyckoff_letter_1),
 (element_2, wyckoff_letter_2),
 ...]

也可以额外记录 multiplicity，但 multiplicity 由 SG + letter 查表得到。

生成 skeleton 时维护状态：

target_formula_counts
remaining_counts
target_sg
valid_wyckoff_letters
used_nonrepeatable_fixed_sites
current_logprob

约束规则：

1. element 选择：
只允许 remaining_counts[element] > 0 的元素。

2. Wyckoff letter 选择：
只允许：
- letter 属于当前 SG；
- multiplicity(SG, letter) <= remaining_counts[element]；
- 不可重复 fixed Wyckoff site 不能重复使用；
- 选择该 letter 后，remaining_counts 仍可闭合。

3. formula closure：
实现 can_close DP / backtracking：
判断剩余元素计数是否能被当前 SG 的 Wyckoff multiplicity 组合闭合。
不能闭合的路径直接剪枝。

4. STOP：
只有 remaining_counts 全部为 0，才允许结束 skeleton。

5. logprob：
每一步候选 element / letter 使用当前模型 logprob 打分。
beam 中累计 logprob。
最终选 top-k skeleton。

推荐参数：
- beam_size = 32
- top_skeletons = 5
- max_sites_per_structure = 合理上限，例如 32 或根据 target formula 自动设定
- 每个 skeleton 生成 1 个或多个 coord/cell continuation
- 最终每个 sample 输出 n=5 candidates

注意：
- 不使用 GT skeleton
- 不使用 GT coords
- 不使用 GT lattice
- 不根据 match 结果筛 skeleton

====================
六、坐标与 cell 生成
====================

对每个 selected skeleton：

1. fixed coordinate：
由 lookup 自动写 FIXED，不采样。

2. free coordinate：
由模型采样。
数值必须合法，归一化到 [0,1)，保留四位小数。
采样失败则重采该 coordinate，最多有限次数。

3. cell：
使用第一阶段修复后的 cell generation。
不能再写死 beta=100。
按晶系做合法投影。
