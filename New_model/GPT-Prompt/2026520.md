你现在是我的科研代码助手。当前任务是基于已有的 symcif_v1 生成结果，新增一套 “SymCIF-v1 + constrained repair / rejection” 后处理流程，用来验证：如果对 SymCIF 生成结果施加晶体学硬约束，能否提升 formula_ok、readable、valid、match@1、match@20。

本任务不改模型、不重新训练、不重新生成样本，只处理已有生成结果。

====================
一、背景
====================

当前已有三组生成评估：

baseline:
- match@20 = 45.00%
- formula_ok = 80.06%
- sg_ok = 87.72%
- readable = 91.47%

symcif_v1:
- match@20 = 35.20%
- formula_ok = 22.62%
- sg_ok = 71.03%
- readable = 74.56%

symcif_v1 的主要问题不是几何 RMSE，而是：
1. 生成文本 schema 不稳定；
2. SymCIF 转标准 CIF 时失败较多；
3. formula closure 很差；
4. SG-Wyckoff-site_symmetry-enumeration 组合有非法或不一致；
5. FIXED/free coordinate 位置可能错误。

目标：
在不改模型的前提下，通过 constrained repair / rejection 改善 symcif_v1 的有效生成率和 exact reconstruction 指标。

====================
二、重要原则
====================

1. 严禁使用 ground-truth CIF 内容来修复生成结果。
2. 修复时只能使用：
   - 当前生成文本；
   - 原始 prompt 中已经给出的条件，例如 target formula、target space group；
   - 预先构建好的 Wyckoff lookup table；
   - 晶体学规则。
3. 不要为了提高 match 偷看 GT 坐标、GT Wyckoff、GT cell。
4. 所有修复必须可解释、可记录。
5. 无法确定的样本直接标记失败，不要强行猜。
6. 第一版以 conservative repair 为主，宁可少修，不要乱修。

====================
四、整体流程
====================

对每一条 symcif_v1 生成结果，按下面步骤处理：

原始 symcif_v1 生成文本
→ schema normalize
→ schema check
→ SG-Wyckoff legality check
→ site_symmetry/enumeration correction
→ FIXED/free coordinate correction
→ formula closure check
→ lattice/cell field check
→ render standard CIF
→ pymatgen read
→ 原有 evaluator 评估

====================
五、具体处理步骤
====================

第 1 步：Schema normalize

目标：修复明显的格式问题，让文本更容易被 parser 读取。

允许做的修复：
1. 去掉多余空行、非法注释、乱码 token。
2. 统一字段名，例如只接受下面这些字段：
   - data_
   - _chemical_formula_sum
   - _symmetry_Int_Tables_number
   - _symmetry_space_group_name_H-M
   - _cell_formula_units_Z
   - loop_
   - _wyckoff_site_index
   - _wyckoff_site_element
   - _wyckoff_site_multiplicity
   - _wyckoff_site_letter
   - _wyckoff_site_symmetry
   - _wyckoff_site_enumeration
   - _wyckoff_free_x
   - _wyckoff_free_y
   - _wyckoff_free_z
   - _cell_length_a/b/c
   - _cell_angle_alpha/beta/gamma
   - _cell_volume
3. 如果 site index 不连续，可以重新编号。
4. 如果一行字段数量多一个或少一个，只有在能明确对齐时才修复。
5. 如果 loop header 缺失或严重错乱，直接标记 schema_invalid。

不允许做的修复：
1. 不要凭空补 Wyckoff site。
2. 不要凭空补元素。
3. 不要凭空补自由坐标。
4. 不要用 GT 文件补字段。

第 2 步：Space group check

从生成文本中解析：
- sg_number
- sg_symbol

检查：
1. sg_number 是否在 1-230。
2. sg_symbol 是否和 sg_number 对应。
3. 如果 sg_number 和 sg_symbol 冲突，优先使用 sg_number，并用 lookup 或 pymatgen 标准 symbol 覆盖 sg_symbol。
4. 如果 sg_number 缺失，但 sg_symbol 存在，则尝试反查 sg_number。
5. 如果两者都缺失，标记 sg_missing。

第 3 步：Wyckoff legality check

对每一个 Wyckoff site 检查：

1. 当前 SG 下是否存在该 Wyckoff letter。
2. 该 letter 的 multiplicity 是否和生成文本中的 multiplicity 一致。
3. 该 letter 的 site symmetry 是否和生成文本中的 site symmetry 一致。
4. 该 letter 的 enumeration 是否和生成文本中的 enumeration 一致。
5. 该 letter 的 free coordinate mask 是否和生成文本中的 FIXED/free 数值位置一致。

修复规则：
1. 如果 sg_number + wyckoff_letter 合法，则优先相信 wyckoff_letter。
2. multiplicity 不一致时，用 lookup 中的 multiplicity 覆盖生成值。
3. site_symmetry 不一致时，用 lookup 中的 site_symmetry 覆盖生成值。
4. enumeration 不一致时，用 lookup 中的 enumeration 覆盖生成值。
5. free/FIXED mask 不一致时：
   - lookup 显示固定的位置，强制改成 FIXED；
   - lookup 显示自由的位置，如果生成的是合法数字，则保留；
   - lookup 显示自由的位置，但生成的是 FIXED 或非数字，则标记 coord_invalid，不要乱补。

如果 wyckoff_letter 在当前 SG 下不存在：
- 第一版直接标记 wyckoff_invalid。
- 不要根据 site_symmetry/enumeration 反推并强行替换，避免过度修复。
- 可以额外记录：是否存在 site_symmetry + enumeration 对应的合法 letter，供后续分析，但第一版不自动替换。

第 4 步：FIXED/free coordinate correction

目标：保证坐标字段符合 Wyckoff template。

规则：
1. 固定坐标位置必须是 FIXED。
2. 自由坐标位置必须是数值。
3. 数值统一保留四位小数。
4. 数值需要映射到 [0, 1) 区间。
5. 如果生成 1.0000，改成 0.0000。
6. 如果生成负数，做 modulo 1。
7. 如果自由坐标缺失或为 FIXED，不要猜，标记 coord_invalid。
8. 如果固定位置生成了数字，直接改成 FIXED。

第 5 步：Formula closure check

这是最重要的一步。

从 prompt 或 split_manifest 中读取 target cell formula，例如：
Na2Cl2

然后根据修复后的 Wyckoff site 计算生成结构的 cell formula：

对每个 site：
element_count[element] += multiplicity

检查：
generated_formula == target_formula

第一版策略：
1. 如果 formula 完全闭合，则通过。
2. 如果 formula 不闭合，直接标记 formula_not_closed，并拒绝该生成样本。
3. 不要第一版就自动增删 site。
4. 不要第一版就自动改 element。
5. 不要第一版就自动改 Wyckoff letter。

原因：
这一步是为了先验证“严格约束过滤”能把低质量生成剔除掉，而不是引入复杂修复导致结果不可解释。

需要记录：
- target_formula
- generated_formula
- element_count_diff
- failed element
- over_count / under_count

第 6 步：Lattice / cell field check

检查是否存在：
- _cell_length_a
- _cell_length_b
- _cell_length_c
- _cell_angle_alpha
- _cell_angle_beta
- _cell_angle_gamma

规则：
1. 六个 cell 参数必须存在。
2. cell length 必须为正数。
3. cell angle 必须在合理范围，例如 30-150 度。
4. 根据 SG 所属晶系，可以做保守修复：
   - cubic: a=b=c, alpha=beta=gamma=90
   - tetragonal: a=b, alpha=beta=gamma=90
   - orthorhombic: alpha=beta=gamma=90
   - hexagonal/trigonal hex setting: a=b, alpha=beta=90, gamma=120
5. 如果生成值和晶系约束冲突，第一版可以按晶系约束投影：
   - 例如 cubic 下 a,b,c 取平均值；
   - 角度改成对应固定角。
6. 如果 cell length 缺失，不要用 GT 补，标记 cell_missing。
7. 如果 cell volume 缺失，可以根据 a,b,c,alpha,beta,gamma 重新计算。
8. 如果 cell volume 和 cell 参数不一致，优先相信 cell 参数并重新计算 volume。

第 7 步：Render standard CIF

对通过上述检查的样本，转成标准 CIF。

渲染规则：
1. 使用修复后的 SG。
2. 使用修复后的 lattice。
3. 使用修复后的 Wyckoff site。
4. 根据 SG + Wyckoff letter + free coords，从 lookup 中还原 representative coordinate。
5. 再写成标准 CIF atom_site loop。
6. 输出后必须能被 pymatgen 读取。
7. 再用 spglib/pymatgen 检查 formula、space group、multiplicity。

第 8 步：Evaluation

使用原有 evaluator 对 symcif_v1_constrained 做评估。

需要和 baseline / raw symcif_v1 对齐：
- same test samples
- same n=20 generations
- same StructureMatcher
- same max_match_sites
- same evaluator 参数

输出指标：
- parse
- repair_success
- to/readable CIF
- formula_ok
- sg_ok
- multiplicity_ok
- valid
- bond score
- match@1
- match@20
- RMSE
- timeout

====================
八、成功标准
====================

这轮实验不是必须超过 baseline，但至少要回答清楚：

1. formula_ok 能不能从 22.62% 拉高到 60% 以上；
2. readable 能不能从 74.56% 拉高到 85% 以上；
3. match@20 能不能从 35.20% 明显提升；
4. 如果 match@20 没提升，主要是因为：
   - 通过约束后剩余样本太少？
   - formula closure 太严格？
   - 模型本身没有学到正确 Wyckoff 骨架？
   - free coordinate / lattice 仍然不准？
   - 还是 evaluator 问题？

如果 constrained postprocess 后，formula_ok/readable 明显提升，但 match@20 仍然不提升，说明问题主要不在格式合法性，而在模型生成的 Wyckoff 骨架和目标结构不一致。

如果 constrained postprocess 后，match@20 明显提升，说明下一步值得实现 dynamic mask，把这些约束前移到采样阶段。
