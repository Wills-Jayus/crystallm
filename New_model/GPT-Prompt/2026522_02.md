新增：

核心函数：
extract_free_params_from_source_coord(
    source_coord,
    orbit_token,
    tolerance=1e-4
) -> dict | None

逻辑：
- OrbitToken 有 canonical representative_expr，例如 (x,0,0)、(x,2x,z)、(x,x,x)
- source CIF 的代表坐标可能不是 canonical representative，例如 SG225 24e source 是 (0,0,z)，而 OrbitEngine canonical 写成 (x,0,0)
- 对 source_coord，遍历 OrbitToken 的 symmetry operations / inverse operations，把 source_coord 映射回 canonical representative frame
- 对每个 mapped_coord，求解 representative_expr 中的自由变量
- 表达式按线性分式处理，支持：
  x, y, z
  -x, -y, -z
  2x, x/2
  x+y 这类如果存在则先标记 unsupported
  固定值 0, 1/2, 1/4, 1/8 等
- 使用 mod 1 fractional coordinate 误差
- 选择 residual 最小且展开 multiplicity 正确的 free_params
- 如果找不到合法 params，记录 failure，不要静默设为 0
