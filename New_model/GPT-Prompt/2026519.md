========================
一、整体实验设计
========================

请准备三组数据格式：

A. Baseline 格式：
使用原始 CrystaLLM 的预处理 CIF 格式，不做结构性改造。

B. CrystalFormer-like 顺序格式：
把 CIF 改成更符合晶体学生成顺序的文本格式：
composition → space group → Wyckoff sites → element → free coordinates → lattice

只包含：
- formula
- space group
- Wyckoff letter
- multiplicity
- element
- free fractional coordinates
- lattice parameters

不加入 site symmetry / enumeration。

C. SymCIF-v1 格式：
在 B 的基础上加入 WyFormer 风格的辅助字段：
- site symmetry
- enumeration

第一轮实验比较：
A vs B vs C

目标是验证：
1. 只改生成顺序是否比原始 CrystaLLM 格式更好；
2. 加入 site symmetry + enumeration 是否进一步提升；
3. 在不改模型结构、不改解码方式的前提下，新数据格式是否能提升 valid / match rate / RMSE。

========================
五、site symmetry + enumeration 要求
========================

第一版不要盲目相信外部 lookup table。

请按这个原则实现：
1. 优先自己基于当前环境的 pymatgen / spglib / pyxtal 构建 lookup table。
2. 可以读取或参考 WyFormer 的 lookup table，但必须做一致性校验。
3. enumeration 定义：
   对于同一个 space group 内，具有相同 site symmetry 的 Wyckoff positions，按照 conventional Wyckoff order 从 0 开始编号。

例如：
如果 SG=225 中：
4a 和 4b 都是 m-3m，则：
4a -> site_symmetry=m-3m, enumeration=0
4b -> site_symmetry=m-3m, enumeration=1

需要保存：
artifacts/wyckoff_lookup.json

格式大致为：
{
  "225": {
    "4a": {
      "multiplicity": 4,
      "letter": "a",
      "site_symmetry": "m-3m",
      "enumeration": 0,
      "free_mask": [false, false, false],
      "template": "..."
    }
  }
}
