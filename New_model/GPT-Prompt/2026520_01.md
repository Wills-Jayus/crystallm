====================
六、Accepted Candidate 定义
====================

主实验使用 conservative accepted 标准：

一个 SymCIF generation 被 accepted，当且仅当满足：

1. SymCIF schema parse 成功；
2. space group 能解析；
3. space group 与 prompt target space group 一致；
4. 每个 Wyckoff letter 在当前 SG 下合法；
5. free/FIXED coordinate mask 合法；
6. formula closure 成功：
   由 Wyckoff multiplicity 汇总得到的 generated cell formula 必须等于 prompt target formula；
7. cell parameters 存在且数值合法；
8. 可以成功 render 成标准 CIF；
9. pymatgen 可以读取标准 CIF。

注意：
- 不要使用 GT CIF 修复生成结果。
- 不要使用 GT Wyckoff、GT 坐标、GT cell。
- prompt 中的 formula 和 space group 可以使用，因为它们是生成条件。
- 不允许为了闭合公式而自动增删 site。
- 不允许为了闭合公式而自动替换 element。
- 不允许用 StructureMatcher 结果反向选择样本。

可选附加版本：
再做一个 strict accepted 标准，在上述基础上要求 valid=True。
这个版本只作为附录，不作为主结论，因为它会改变候选分布。

====================
七、评估方式
====================

对每个 test sample：

1. 生成最多 200 个 raw candidates；
2. 依次筛选 accepted candidates；
3. 取前 20 个 accepted candidates；
4. 对这 20 个 accepted candidates 执行原 evaluator；
5. 如果其中任意一个与 GT match，则该 sample 的 match@20 = true；
6. RMSE 取该 sample 中 matched candidates 的最小 RMS；
7. 如果 accepted candidates 少于 20，也照常评估，并记录 accepted_count。

最终输出两套指标：

A. raw-attempt 口径：
分母是所有 raw generations。
用于观察生成器本身有多少比例能产生 accepted candidate。

B. accepted@20 口径：
每个 sample 最多看前 20 个 accepted candidates。
用于回答：如果 constrained decoding 能稳定产生合法候选，SymCIF 的 match@20 潜力是多少。

====================

如果 extended rejection 结果为：

- accepted@20 match@20 明显高于 35.20%，例如达到 40% 以上：
  说明 formula/site closure 是主要瓶颈，constrained decoding 有必要继续做。

  说明 SymCIF 主线非常有希望，可以优先投入。

  说明即使过滤出 formula-closed/readable 候选，结构骨架仍不够对，下一步应该重新设计 SymCIF-v2 或训练目标，而不是只做解码约束。

====================
十四、关于后续 constrained decoding 的设计备注
====================

本任务只做 extended rejection，不实现 constrained decoding。

但请在最终报告最后写一个简短设计建议：

后续真正 constrained decoding 应该参考 CrystalFormer 的方式，把非法选择在采样时 mask 掉，而不是事后 repair。

原则：
1. 优先修改 tokenizer / vocabulary，让 element、Wyckoff letter、STOP、cell section 等关键字段成为可控 token。
2. 生成 Wyckoff site 时维护 remaining formula counts。
3. 只允许采样 target formula 中还剩余的元素。
4. 只允许采样当前 SG 下合法 Wyckoff letter。
5. 只允许采样 multiplicity 不超过 remaining count 的 Wyckoff letter。
6. 只有 remaining counts 全部归零时，才允许结束 Wyckoff loop 并进入 cell section。
7. multiplicity、site_symmetry、enumeration 应尽量由 lookup 自动填，不要让模型自由生成。
8. FIXED/free coordinate mask 应由 lookup 决定，固定坐标不让模型生成。
9. 不要引入复杂的事后补偿方案；优先在采样阶段保证合法性。
