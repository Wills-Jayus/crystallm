# Opentry-2 Model-Side Training Strategy

日期：2026-06-11 UTC

## Objective

在不使用 test label、GT-W/A、source CIF fallback、target row_count 或 StructureMatcher oracle rerank 的前提下，通过模型侧训练提高 MP-20 / MPTS-52 的 `match@1`、`match@5`、`match@20`，并最终让至少两个 match 指标比同 GT-SG CrystaLLM basemodel 高 5pp。

## Current Diagnosis

历史报告反复指向同一瓶颈：

- GT-WA 已知时，e07/e08 geometry 在 MP-20 clean_val 已达到 `match@5=85-87%`、RMSE `~0.043`，geometry 不再是优先方向。
- Full generation 的 K20 相对 K5 增益很小，说明候选池本身缺少正确 W/A，而不是简单 top-k 不够。
- MPTS-52 rows>=7 占近一半，pure SymCIF rows>=7 match@5 只有约 `13-14%`。
- skeleton@5 有时显著高于 WA@5，说明 row skeleton 有部分覆盖，但 element assignment / exact W/A multiset / ranking 失败。

## Candidate Training Routes

| route | scope | expected benefit | risk |
|---|---|---|---|
| Neural W/A reranker | train on train split positive true W/A vs train-prior negatives, select/rank candidates generated by existing v5 runner | Fastest; preserves existing renderer and e08 geometry; can target rank-1 + top-k diversity | If candidate pool lacks correct W/A, reranking alone cannot fix top-k coverage |
| Skeleton-first proposal model | predict row_count and Wyckoff skeleton from formula + GT-SG, then exact-cover element assignment | Directly targets rows>=7 coverage | Needs careful beam constraints and more implementation work |
| Element-assignment model | given formula + GT-SG + skeleton, rank element-to-row assignments | Addresses skeleton-hit vs WA-hit gap | Requires generating skeleton candidates and many negatives |
| SFT over structured W/A sequences | train autoregressive sequence model over W/A rows, with complex oversampling | More model-side and generative | Slower to integrate into existing exact-cover runner |
| DPO / preference training | learn preferences from val/train candidate metrics or proxy labels | Could improve rank quality | Must avoid test labels; StructureMatcher-derived labels should be train/val only and separated from final test |
| Distillation from CrystaLLM candidates | use CrystaLLM-generated matched train candidates as additional supervision | Could import useful top-k coverage | Needs strict train-only generation/eval; expensive and method becomes partially CrystaLLM-assisted |

## First Route

Start with a train-only neural W/A reranker/proposal for MPTS-52:

1. Build train/val candidate tables using existing v5 candidate generators.
2. Label positives from the structured train/val target W/A keys, not StructureMatcher test labels.
3. Train with complex weighting for rows>=7 / atom>=12.
4. Add diversity-aware inference: keep E04/E13 rank-1 candidate first unless model confidence says otherwise; fill top-k with high-scoring unique W/A candidates that differ in skeleton/assignment.
5. Evaluate on val first. Only freeze and run test if val improves both top-1 and top-k gates.

## Metrics To Track

- overall `match@1`, `match@5`, `match@20`, `RMSE@1`, `RMSE@5`, `RMSE@20`;
- rows>=7 and atom>=12 `match@5/20`;
- WA@5/20, skeleton@5/20;
- top-k unique W/A count and unique skeleton count;
- rank-1 preservation relative to v5_e1/E13.

## Strict Non-Leakage Rules

- Train on train split only.
- Use val only for route/model selection.
- Use test once per frozen configuration.
- Do not read test evaluator `match_ok`, `rms`, `target_wa`, or GT CIF for training/ranking.
- Do not use source CIF fallback or target row_count at inference.
- Store all new artifacts in `model/New_model/opentry_2`.
